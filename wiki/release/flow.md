# Release flow

## Bump → tag → publish

```bash
bun run bump:app <version>           # 0.6.0 or 0.6.0-rc.0
git add -u && git commit -m "chore: bump version to <version>"
bun run release:check                # ~3min, full gate (9 steps)
git tag v<version>
git push origin main --tags          # tag triggers build-desktop.yml
```

The version files MUST be in sync with the tag. `bump:app` updates
all of:
- `apps/desktop/package.json`
- `apps/desktop/src-tauri/tauri.conf.json`
- `apps/desktop/src-tauri/Cargo.toml`
- `apps/desktop/src-tauri/Cargo.lock` (via `cargo check` after the bump)

Never edit those files manually.

## Pre-tag checklist

| Gate | Command | When |
|---|---|---|
| Fast smoke | `bun run release:smoke` | every commit before tag |
| Full check | `bun run release:check` | once before the tag |
| Manual UX | `bun run dev:app` | smoke-test the actual flow that changed |

`release:check` runs:

1. typecheck (all workspaces)
2. `bun test`
3. `cargo test --lib` + perf gates (`--ignored`)
4. vitest (UI render)
5. cargo-llvm-cov summary
6. conversion benchmarks (vs baseline)
7. StrykerJS mutation testing on packages/core (~2 min)
8. E2E specs (spawns `tauri dev`, runs specs, tears down)
9. dependency audits (`bun audit` + `cargo audit`, advisory-only)

## Auto-update endpoint

```
https://github.com/djalmajr/asciimark-releases/releases/latest/download/latest.json
```

Generated by `build-desktop.yml`'s "Normalize assets and generate
latest.json" step. Each platform entry needs both `signature`
(content of the `.sig` file) and `url`. Missing signatures = empty
`platforms` map = clients can't update.

## What the public release contains

| Platform | Update asset (auto-update) | Installer (first install) |
|---|---|---|
| macOS arm64 | `*.app.tar.gz` | `*.dmg` |
| macOS x64 | `*.app.tar.gz` | `*.dmg` |
| Linux x64 | `*.AppImage.tar.gz` | `*.AppImage` or `*.deb` |
| Windows x64 | `*.nsis.zip` | `*.msi` or `*-setup.exe` |

Each update asset has a sibling `.sig` (ed25519 / minisign). Without
it the client refuses to install.

## Hard rules

- **NEVER** trigger pipelines manually without explicit user
  confirmation.
- **NEVER** create or modify tags without confirmation.
- **NEVER** modify workflows without confirmation.
- **NEVER** change the `pubkey` in `tauri.conf.json` without
  understanding the impact (every existing client loses auto-update
  permanently if the key rotates).
- **NEVER** read or store the private key (`~/.tauri/asciimark.key`)
  in tools / AI / pastebins.
