= Usage

Daily workflows that get the most out of AsciiMark. New here? Start
with xref:getting-started.adoc[Getting Started] first.

toc::[]

== Tabs

=== Preview vs pinned

Single-click in the file tree opens the file in the *preview* slot
— the tab title shows in italic. Click another file and the
preview is replaced. Double-click, drag, or just start editing to
*pin* the tab; pinned tabs stay until you close them.

The invariant: at most one preview tab per pane. Middle-click and
"Open in New Tab" always create a pinned tab.

=== Reorder and move

Drag tabs to reorder within a pane. Drag onto the splitter to move
to the other pane (creates the second pane if needed).

== Split panes

`Cmd/Ctrl+\` toggles a second pane. Each pane has its own tab list,
editor mode (Edit / Edit & Preview / Preview), scroll position,
and TOC. The active pane gets full opacity; the inactive one fades
slightly so the focus indicator reads at a glance.

`Cmd/Ctrl+1` and `Cmd/Ctrl+2` focus pane 1 and pane 2.

== TOC and References

The right gutter is segmented into two tabs:

* *Summary* — the rendered document's table of contents.
* *References* — every other workspace file that links to the
  current doc (markdown link, asciidoc xref, asciidoc include).
  The count badge tells you upfront whether the active doc is
  referenced.

Both tabs stay mounted regardless of which is visible — switching
is instant.

== Reader Mode

`Cmd/Ctrl+.` (or *View → Toggle Reader Mode* in the Command
Palette) collapses every chrome surface and centers the preview
at a comfortable reading width. Press the same shortcut to come
back.

[#shortcuts]
== Shortcuts cheatsheet

[cols="1,3"]
|===
| `Cmd/Ctrl+P` | Quick Open — fuzzy file picker
| `Cmd/Ctrl+Shift+P` | Command Palette — every action
| `Cmd/Ctrl+Shift+O` | Go to Symbol — current file
| `Cmd/Ctrl+Alt+O` | Go to Symbol in Workspace
| `Cmd/Ctrl+Shift+F` | Find in Files
| `Cmd/Ctrl+\` | Split editor (toggle)
| `Cmd/Ctrl+1` / `Cmd/Ctrl+2` | Focus pane 1 / 2
| `Cmd/Ctrl+.` | Toggle Reader Mode
| `Cmd/Ctrl+/` | Show all shortcuts
|===

== Status bar

The bottom strip shows the active file's path on the left and a
*words · reading time* pill on the right. Reading time counts
against the rendered HTML, so AsciiDoc `include::` and Markdown
transcludes are factored in correctly.
