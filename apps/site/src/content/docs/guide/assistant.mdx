A built-in assistant sits in the right panel, aware of the document you're editing. Open or focus it with <code>Cmd/Ctrl+L</code>, or from the right-panel tabs. Nothing is sent anywhere until you connect a provider — see <strong>AI Providers</strong> below.

## Plan, Build, and Ask modes

<strong>Plan</strong> keeps the assistant read-only, so it can outline an approach without changing anything. <strong>Build</strong> lets it run tools automatically. <strong>Ask</strong> keeps the same tools available, but shows an approval window before each tool call. Switch modes from the composer.

## Context: selections, files, and mentions

The active document is always in context. Add more by selecting text in the editor, selecting shapes on an Excalidraw canvas, or typing <code>@</code> to mention another file or folder. Context chips show exactly what the assistant can see.

## Slash commands

Type <code>/</code> in the composer to insert a reusable prompt template. Templates can be built in, global, or project-specific, and may take arguments.

## Multiple sessions and history

Keep several conversations open at once as tabs in the right panel. Past sessions are grouped by date and searchable; you can fork, archive, restore, or delete them.

## What the assistant can do

With a provider connected, the assistant can use a set of tools, each gated by your approval when it changes anything:

- <strong>Read the active document</strong> — including the outline of an Excalidraw canvas.
- <strong>Search the workspace</strong> — by keyword, or by meaning when indexing is enabled.
- <strong>Propose edits</strong> — shown as a diff you can accept or reject.
- <strong>Generate and edit diagrams</strong> — see <strong>Diagrams</strong> for editable, AI-generated Excalidraw.
